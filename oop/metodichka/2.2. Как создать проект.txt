Для компиляции нескольких файлов исходного кода и сборки из них одного
исполняемого файла необходимо создать проект. Структура проекта (с
учетом результатов предыдущего этапа) будет следующей:

                                |========|
                          +-----*        |
                          |     |        |
                          |     | list.h |
                          |     |        |
                          |  +--*        |
                          |  |  |========|
        ..................|..|..................
        .                 |  |                 .
        .     |========|  |  |  |========|     .
        .     |        |<-+  +->|        |     .
        .     |        |        |        |     .
        .     | main.c |        | list.c |     .
        .     |        |        |        |     .
        .     |        |        |        |     .
        .     |========|        |========|     .
        .                                      .
        .  файлы, включаемые в состав проекта  .
        ........................................

В Borland C/C++ 3.1 проект создается следующим образом (на примере
файлов для лабораторной работы № 2):

1. Создать папку, в которой будут находиться файлы проекта. Имя папки
   Должно быть коротким (до 8 символов) и не содержать русских букв и
   пробелов (имена DOS). Рекомендуется и в дальнейшем для каждого
   проекта создавать отдельную папку, руководствуясь этим же правилом.

2. Скопировать в эту папку все файлы, которые будут входить в состав
   проекта (например, файлы list.c, list.h, main.c).

3. В среде Borland C/C++ сделать эту папку текущей (команда меню
   File->Change dir).

4. Создать проект (Project->Open). В диалоговом окне следует убедиться,
   что проект создается именно в приготовленной для этого папке (п.1).
   Следует указать имя проекта и, если в выбранной папке проект с таким
   именем не существует, он будет создан.

5. В автоматически появившемся окне Project (при необходимости его можно
   вызвать командой Window->Project) при помощи клавиши Insert (или
   команды Project->Add item) в состав проекта добавляются модули. В
   проект должны быть включены только сами модули, а не их заголовочные
   файлы (например list.c и main.c включаются, а list.h - нет).

      Примечание. При необходимости модули могут быть удалены из проекта
      клавишей Delete в окне Project или командой Project->Delete item.

6. Сохранить настройки проекта (Options->Save), включив при этом все
   опции. После этого в папке будет создан файл проекта с расширением
   prj и файлы с настройками среды. Необходимость этого обусловлена
   возможным отсутствием у пользователя прав на запись в папки среды
   программирования и соответствующими ее настройками, при которых
   файлы проекта и настроек автоматически не сохраняются.

После того, как проект будет создан, такие команды как запуск (Run->Run)
относятся к проекту целиком и приводят к компиляции всех модулей проекта
и запуску целевого исполняемого файла, который имеет то же имя, что файл
проекта, но расширение exe. Чтобы продолжить работу с проектом, его
необходимо открыть командой Project->Open (не File->Open!), выбрав папку
и указав имя проекта.

При переносе проекта с одного компьютера на другой, после его открытия
может понадобиться указать правильные пути к стандартным библиотекам и
их заголовочным файлам (команда Options->Directories, параметры
Library path и Include path), если папки, в которые установлена среда,
различаются на разных компьютерах.

При необходимости проект может быть скопирован целиком из одной папки в
другую с сохранением его работоспособности (например, при выполнении
следующей лабораторной работы), т.е. копируются либо все файлы проекта
в другую папку, либо сама папка, их содержащая.
